    Configurações do codificador

    Codificador de hardware Selecione qual codificador de GPU usar para a transcodificação. A opção “Auto” utilizará o melhor disponível.
    Resolução máxima Limitar a resolução da saída transcodificada. Quanto menor, mais rápido e mais compatível.
    Predefinição de qualidade Uma qualidade superior consome mais largura de banda e recursos da CPU/GPU.
    Predefinição de mixagem de áudio HO áudio multicanal (5.1/7.1) é reduzido a estéreo para reprodução no navegador.

    Ampliação

    Ativar upscaling Aumente a resolução de conteúdos de baixa resolução para uma resolução mais alta durante a transcodificação.
    Metodologia de alto padrãod O hardware é mais rápido; o software (Lanczos) é de melhor qualidade.
    Público-alvo de alto padrão Resolução alvo para a saída com aumento de resolução.

    Processamento de fluxo

    Transcodificação automática (Smart) Detecta automaticamente os codecs de streaming e só transcodifica/remuxa quando necessário. Acrescenta um atraso de sondagem de aproximadamente 1 a 3 segundos na primeira reprodução.
    Forçar transcodificação de áudio Transcodifique o áudio para AAC para garantir a compatibilidade com navegadores. Ative esta opção se o vídeo for exibido, mas não houver áudio (corrige problemas com Dolby/AC3/EAC3).
    Forçar transcodificação de vídeo Forçar a transcodificação completa do vídeo (legendas incorporadas, vídeo incompatível). Utiliza o codificador de hardware.
    Forçar Remux RO emux converte fluxos para o formato MP4. Ative essa opção para fluxos .ts não processados provenientes de middleware de IPTV, como o m3u-editor ou o dispatchar.
    Formato de saída do fluxo Formato de arquivo para transmissões do Xtream. Experimente o TS se tiver problemas de buffering.

    Configurações HTTP

    User-Agent User-Agent HTTP para solicitações de streaming. Experimente o VLC se o seu provedor bloquear o Chrome.
    User-Agent personalizado Insira sua própria string de User-Agent.

    Rede

    Forçar proxy de backend Encaminhe todos os streams pelo servidor para contornar as restrições CORS. Ative esta opção se os streams não forem reproduzidos diretamente.
